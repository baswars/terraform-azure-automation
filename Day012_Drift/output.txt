Terraform Drift:

Terraform drift refers to a situation where the actual infrastructure in Azure differs from what is defined in the Terraform configuration or stored in the state file.”
“This typically happens when someone makes manual changes in the Azure portal or through another tool instead of using Terraform. As a result, Terraform’s state becomes out of sync with the real infrastructure

#### How to Detect Drift
We can detect drift using terraform plan. When we run this command, Terraform compares the current state file with the actual infrastructure in Azure and shows the differences.”
If there is drift, Terraform will show changes even though we didn’t modify the code.

How to Resolve Drift

👉 “There are mainly two ways to resolve drift depending on the situation.”

👉 “First, if the manual changes are not required, we can run terraform apply to bring the infrastructure back to the desired state defined in the code.”

👉 “Second, if the manual changes are valid, we should update the Terraform configuration accordingly and then run terraform apply so that the state file matches the actual infrastructure.”

👉 “In some cases, if the resource is not in the state file, we can also use terraform import to bring existing resources under Terraform management.”
