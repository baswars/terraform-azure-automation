Workspace:

👉 “In Terraform, a workspace is a feature that allows us to manage multiple environments using the same Terraform configuration. Instead of writing separate code for environments like dev, QA, or production, we can use workspaces to maintain separate state files for each environment.”

👉 “Each workspace keeps its own state file, so even though the code is the same, the infrastructure created is different and isolated. For example, if I create a resource in the ‘dev’ workspace, it will not affect the ‘prod’ workspace.”

👉 “We can switch between environments using commands like terraform workspace select dev, and Terraform will automatically use the corresponding state file.
